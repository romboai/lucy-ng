Add a new section `## 5. Inventory Validation Protocol` inside the existing `<domain_knowledge>` block (after section 4). This section contains:

**A. Inventory Extraction**

Document how to extract the inventory from an LSD file using Bash/Grep:

```bash
# Extract inventory JSON from LSD file
grep -A1000 "=== CONSTRAINT INVENTORY v1 ===" compound.lsd | grep -B1000 "=== END CONSTRAINT INVENTORY ===" | grep "^; " | sed 's/^; //' | tr '\n' ' '
```

Note: The inventory is a JSON object embedded as `;`-prefixed comment lines between delimiters, as defined in 43-RESEARCH.md Pattern 1. Do NOT use the key-value fallback format (Pitfall 4). The LSD parser compatibility test in 43-01 confirms JSON characters in `;` comments are safe. The Devils-Advocate reads the inventory but NEVER modifies it (read-only agent).

If no inventory block found (iteration 1 before Phase 43, or legacy files): fall back to existing diff protocol (count-based comparison). Flag WARNING: "No constraint inventory found — using legacy validation."

**B. Three-Check Reconciliation Protocol**

For each constraint type, perform three checks:

**Check 1: Inventory Accurate?** (inventory counts match actual LSD file)

| Inventory Field | Grep Command | Must Match |
|----------------|--------------|------------|
| `mult_count` | `grep -c "^MULT" compound.lsd` | Exact |
| `hsqc_count` | `grep -c "^HSQC" compound.lsd` | Exact |
| `hmbc_total` | `grep -c "^HMBC" compound.lsd` | Exact |
| `deff_not_patterns` (array length) | `grep -c "^DEFF NOT" compound.lsd` | Exact |
| `syme_pairs` (array length) | `grep -c "^SYME" compound.lsd` | Exact |
| `bond_constraints` (array length) | `grep -c "^BOND" compound.lsd` | Exact |
| `grouped_hmbc` (array length) | `grep -c "^HMBC (" compound.lsd` | Exact |

If any mismatch: flag CRITICAL — "Inventory claims N {type} but file has M. Inventory is inaccurate."

**Check 2: No Regression?** (current counts >= previous iteration counts)

Compare current inventory against previous iteration's inventory:

| Field | Expected | Alert If |
|-------|----------|----------|
| `mult_count` | Identical | Any change -> CRITICAL |
| `hsqc_count` | Identical | Any change -> CRITICAL |
| `hmbc_total` | >= previous | Decrease -> CRITICAL |
| `deff_not_patterns` length | >= previous | Decrease -> CRITICAL |
| `syme_pairs` length | >= previous | Decrease -> CRITICAL |
| `bond_constraints` length | >= previous | Decrease -> CRITICAL (unless advisory) |
| `grouped_hmbc` length | >= previous | Decrease -> CRITICAL |

**Check 3: Content Preserved?** (every item in previous arrays exists in current)

For array fields (`deff_not_patterns`, `bond_constraints`, `syme_pairs`, `grouped_hmbc`, each batch in `hmbc_batches`): verify every string in the previous array appears in the current array. Missing item = CRITICAL.

**C. Detection Coverage Check**

Review `pending_from_detection` array:
- If any item has been pending for 3+ iterations (compare iteration numbers): flag WARNING — "Detection result '{item}' pending since iteration N, now iteration N+3. Review whether to apply or mark not_applicable."
- This catches Bug 2 (signal grouping not applied) and Bug 5 (detection constraints not translated).

**D. Updated Bug Checklist Integration**

Update the existing v3.0 UAT Bug Checklist (Section 2) to reference inventory validation:

- Bug 1 (DEFF NOT dropped): Now use inventory `deff_not_patterns` array as source of truth. Check 1 validates inventory accuracy. Check 2 validates no regression. This catches drops at ANY iteration (not just after iteration 1).
- Bug 2 (signal grouping not applied): Now checked via `pending_from_detection` array. If grouping detected but never applied, it stays in pending and triggers WARNING after 3 iterations.
- Bug 3 (grouped notation dropped): Now use inventory `grouped_hmbc` array. Check 2 validates count never decreases.
- Bug 5 (no constraints from detection): Now checked via `applied_from_detection` and `pending_from_detection` arrays. If detection ran but nothing was applied or pending, flag INFO.

Add a note to each bug: "With inventory: check inventory field. Without inventory (legacy): fall back to grep-based counting."

**Formatting guidelines:**
- Keep the new section concise (~50-70 lines). Match existing formatting style.
- Use tables for reconciliation checks (consistent with existing Section 1 tables).
