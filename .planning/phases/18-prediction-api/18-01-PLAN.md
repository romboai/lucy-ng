# Phase 18-01: Database-Backed Prediction API

## Objective

Update `C13Predictor` to use the database `hose_stats` table for shift predictions with radius fallback (6â†’1), replacing the in-memory JSON lookup table as the primary prediction backend.

## Context

**Dependencies:**
- Phase 17 complete: 7,890,374 HOSE statistics in database
- `hose_stats` table has: hose_code, radius, mean, std, count, m2
- `DatabaseManager.get_hose_stats(hose_code, radius)` returns `HOSEStatsRecord`

- `C13Predictor` uses `HOSELookupTable` (in-memory JSON, returns raw shift lists)

**Target State:**
- `C13Predictor` can use either JSON table OR database
- Database backend is O(1) lookup with pre-computed stats
- CLI defaults to database when available

### Task 2: Create DatabaseHOSELookup Adapter

**Goal:** Create adapter class that queries database for HOSE statistics.

**Files:** `src/lucy_ng/prediction/db_lookup.py` (new)

**Steps:**
1. Create `DatabaseHOSELookup` class:
   ```python
   class DatabaseHOSELookup:
       def __init__(self, db: DatabaseManager):
           self._db = db

       def lookup_stats(self, hose_code: str, radius: int) -> HOSEStatsResult | None:
           record = self._db.get_hose_stats(hose_code, radius)
           if record is None:
               return None
           return HOSEStatsResult(mean=record.mean, std=record.std, count=record.count)

       def has_code(self, hose_code: str, radius: int) -> bool:
           return self._db.get_hose_stats(hose_code, radius) is not None
   ```

2. Add `from_db_path` classmethod for convenience:
   ```python
   @classmethod
   def from_db_path(cls, db_path: Path) -> "DatabaseHOSELookup":
       db = DatabaseManager(db_path)
       return cls(db)
   ```

3. Export from `prediction/__init__.py`

### Task 3: Update C13Predictor for Dual-Backend Support

**Goal:** Modify predictor to work with either lookup backend.

**Files:** `src/lucy_ng/prediction/predictor.py`

**Steps:**
1. Change `__init__` to accept either backend type:
   ```python
   def __init__(
       self,
       lookup: HOSELookupTable | DatabaseHOSELookup,
       max_radius: int = 6,
       min_radius: int = 1,
   ):
   ```

2. Update `_predict_for_atom()` to use `lookup_stats()` with radius:
   - For each radius from max to min:
     - Generate HOSE code at that radius
     - Call `lookup.lookup_stats(hose_code, radius)`
     - If result found, create prediction

3. Update `_create_prediction()` to use `HOSEStatsResult`:
   - Use `stats.mean` as shift (instead of median of raw list)
   - Use `stats.std` directly
   - Use `stats.count` for match_count
   - Set min_shift=max_shift=mean (stats backend doesn't have these)

4. Add `from_database()` classmethod:
   ```python
   @classmethod
   def from_database(cls, db_path: Path, **kwargs) -> "C13Predictor":
       lookup = DatabaseHOSELookup.from_db_path(db_path)
       return cls(lookup, **kwargs)
   ```

5. Keep `from_table_file()` for backward compatibility

### Task 4: Update CLI with Database Option

**Goal:** Add `--db` option to `predict c13` command.

**Files:** `src/lucy_ng/cli/predict.py`

**Steps:**
1. Add default database path constant:
   ```python
   DEFAULT_DB_PATH = Path("data/reference/lucy-ng-derep.db")
   ```

2. Add `--db` option to `predict_c13` command:
   ```python
   @click.option(
       "--db",
       type=click.Path(exists=True),
       default=None,
       help="Path to SQLite database with HOSE stats (preferred over JSON table).",
   )
   ```

3. Update command logic to prefer database:
   ```python
   if db:
       predictor = C13Predictor.from_database(Path(db), max_radius=max_radius)
   elif table:
       predictor = C13Predictor.from_table_file(Path(table), max_radius=max_radius)
   else:
       # Try database first, then table
       if DEFAULT_DB_PATH.exists():
           predictor = C13Predictor.from_database(DEFAULT_DB_PATH, max_radius=max_radius)
       elif (table_path := find_lookup_table()):
           predictor = C13Predictor.from_table_file(table_path, max_radius=max_radius)
       else:
           raise error
   ```

4. Update help text and error messages
