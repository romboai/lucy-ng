# Phase 2.1: 1D Carbon Dereplication — Summary

## Outcome

**Status**: Complete
**Date**: 2026-01-09

Successfully implemented 13C NMR dereplication against nmrshiftdb reference data with multiple matching modes and fuzzy tolerances.

## What Was Built

### Core Components

1. **NMRShiftDB Loader** (`src/lucy_ng/dereplication/nmrshiftdb.py`)
   - `HydrogenCount` enum for DEPT classification (C/CH/CH2/CH3)
   - `CarbonSignal` and `NMRShiftDBEntry` dataclasses
   - SD file parsing via RDKit with molecular formula indexing
   - Multiplicity code parsing (S/D/T/Q → hydrogen count)

2. **Peak Picker** (`src/lucy_ng/processing/peak_picker.py`)
   - `SimplePeakPicker` with threshold-based detection
   - Uses scipy.signal.find_peaks
   - Optional noise-estimation variant with SNR threshold

3. **Spectrum Matcher** (`src/lucy_ng/dereplication/matcher.py`)
   - Two matching modes: `SHIFTS_ONLY`, `DEPT_ENHANCED`
   - `MatchingConfig` with variable tolerances by region:
     - Aliphatic (0-80 ppm): 0.8 ppm
     - Aromatic (100-160 ppm): 1.2 ppm
     - Carbonyl (160-220 ppm): 1.5 ppm
   - Overlap-adjusted scoring using molecular formula carbon count
   - Detailed match tracking (matched, unmatched, deviations)

4. **Dereplication Service** (`src/lucy_ng/dereplication/service.py`)
   - Three entry points:
     - `dereplicate_from_spectrum()`: Full automation
     - `dereplicate_from_peaks()`: With optional DEPT info
     - `dereplicate_from_shifts()`: Simple ppm list
   - Auto-detects DEPT mode when hydrogen counts provided

### Test Coverage

18 tests in `tests/test_dereplication.py`:
- NMRShiftDB loader: spectrum parsing, DEPT codes, carbon counting
- Peak picker: threshold-based picking, sensitivity tests
- Spectrum matcher: shifts-only, DEPT-enhanced, variable tolerance
- Dereplication service: all three entry points, edge cases
- Integration tests: full pipeline with Ibuprofen data (conditional)

## Key Decisions Made

| Decision | Rationale |
|----------|-----------|
| Multi-mode matching | Support different data quality levels (shifts-only vs DEPT-enhanced) |
| Overlap-adjusted scoring | Account for fewer peaks than expected carbons due to overlap |
| Variable tolerance by region | Aliphatic shifts more consistent, carbonyl more variable |
| Greedy matching algorithm | Simple and effective for non-overlapping reference sets |
| Three service entry points | Flexibility for different use cases (AI-driven, manual, automated) |
