### nmrshiftdb SD File Format

Each record contains:
- MOL block with structure
- `<INChI>` and `<INChI key>` fields
- `<Spectrum 13C 0>` field with format: `shift;multiplicity;atom_index|shift;multiplicity;atom_index|...`
  - shift: chemical shift in ppm (e.g., `17.6`)
  - multiplicity: J-coupling pattern code (e.g., `0.0Q` = quartet = CH3, `0.0T` = triplet = CH2, `0.0D` = doublet = CH, `0.0S` = singlet = quaternary C)
  - atom_index: reference to MOL atom numbering

**Multiplicity → Hydrogen count mapping** (from coupled 13C / DEPT, n+1 rule):
- `S` (singlet) → C (quaternary, 0 attached H)
- `D` (doublet) → CH (methine, 1 attached H)
- `T` (triplet) → CH2 (methylene, 2 attached H)
- `Q` (quartet) → CH3 (methyl, 3 attached H)

Molecular formula can be derived from MOL block or InChI.

## Real-World NMR Challenges

### Why observed peaks ≠ expected carbons

1. **Overlapping signals**: Equivalent carbons or accidental degeneracy produce single peaks
2. **Quaternary carbons**: Long T1 relaxation → weak or missing signals
3. **Detection threshold**: Low-intensity peaks below noise floor
4. **Solvent artifacts**: Suppression can affect nearby peaks

### Why chemical shifts vary

1. **Solvent effects**: CDCl3 vs DMSO-d6 can shift peaks 1-3 ppm
2. **Temperature**: Typically 0.01-0.1 ppm/°C
3. **Concentration**: Aggregation effects, especially for polar compounds
4. **Reference calibration**: TMS vs solvent referencing
5. **pH**: Ionizable groups can shift 5+ ppm

## Matching Strategy

### Input Modes

| Mode | Input | Discriminating Power | Use Case |
|------|-------|---------------------|----------|
| **shifts_only** | List of δC values | Low | Quick screening, no DEPT data |
| **dept_enhanced** | Shifts + H-count (CH3/CH2/CH/C) | High | Standard case with DEPT-135 |
| **annotated** | Shifts + H-count + atom labels | Highest | Full analysis with assignments |

### Scoring Algorithm

**Problem with naive scoring**: `score = (matched/observed) × (matched/reference)` penalizes overlapping peaks unfairly.

**Solution**: Account for expected overlap using molecular formula.

```python
def calculate_score(
    matched_peaks: int,
    observed_peaks: int,
    reference_peaks: int,
    expected_carbons: int,  # from molecular formula
    dept_matches: int | None = None,  # if DEPT mode
) -> float:
    # Overlap factor: if 13C has 13 carbons but 10 observed peaks,
    # we expect ~3 overlapping and shouldn't penalize reference misses as harshly
    overlap_factor = min(1.0, observed_peaks / expected_carbons)

    # How well do observed peaks match something?
    coverage = matched_peaks / observed_peaks if observed_peaks > 0 else 0

    # How much of the reference did we match?
    # Adjusted by overlap expectation
    adjusted_reference = reference_peaks * overlap_factor
    reference_coverage = matched_peaks / adjusted_reference if adjusted_reference > 0 else 0

    # Base score: geometric mean of coverages
    base_score = (coverage * reference_coverage) ** 0.5

    # DEPT bonus: if multiplicities also match, boost score
    if dept_matches is not None and matched_peaks > 0:
        dept_ratio = dept_matches / matched_peaks
        base_score *= (0.8 + 0.2 * dept_ratio)  # Up to 20% boost

    return base_score
```

```python
from enum import IntEnum

class HydrogenCount(IntEnum):
    """Number of attached hydrogens (DEPT classification)."""
    C = 0   # Quaternary carbon
    CH = 1  # Methine
    CH2 = 2 # Methylene
    CH3 = 3 # Methyl

@dataclass
class CarbonSignal:
    """A single 13C signal with shift and optional DEPT info."""
    shift: float  # chemical shift in ppm
    hydrogen_count: HydrogenCount | None = None  # from DEPT, None if unknown

@dataclass
class NMRShiftDBEntry:
    """A compound entry from nmrshiftdb with 13C spectrum."""
    nmrshiftdb_id: int
    molecular_formula: str
    carbon_count: int  # number of carbons from formula
    inchi: str
    inchi_key: str
    signals: list[CarbonSignal]  # 13C signals with DEPT info

    @property
    def shifts(self) -> list[float]:
        """Return just the shift values for simple matching."""
        return [s.shift for s in self.signals]

class NMRShiftDBLoader:
    """Loader for nmrshiftdb SD file data."""

    def __init__(self, sd_file_path: str | Path):
        """Initialize with path to SD file."""

    def load(self) -> list[NMRShiftDBEntry]:
        """Parse SD file and return all entries with 13C spectra."""

    def get_by_formula(self, formula: str) -> list[NMRShiftDBEntry]:
        """Get all entries matching a molecular formula."""

    @staticmethod
    def parse_spectrum_field(field: str) -> list[CarbonSignal]:
        """Parse nmrshiftdb spectrum field format.

        Format: 'shift;mult;atom|shift;mult;atom|...'
        Example: '17.6;0.0Q;1|22.4;0.0T;2|...'

        Multiplicity codes (n+1 rule): S=C(0H), D=CH(1H), T=CH2(2H), Q=CH3(3H)
        """

    @staticmethod
    def count_carbons(formula: str) -> int:
        """Extract carbon count from molecular formula."""
```

Implementation notes:
- Use RDKit to parse SD file (rdkit.Chem.SDMolSupplier)
- Extract molecular formula from MOL using rdkit.Chem.rdMolDescriptors.CalcMolFormula
- Parse `<Spectrum 13C 0>` field to extract shifts AND DEPT multiplicity
- Map nmrshiftdb multiplicity codes (S/D/T/Q) to HydrogenCount enum
- Extract carbon count from molecular formula for overlap calculations
- Index by molecular formula for fast lookup

**Acceptance**: Can load SD file and filter by molecular formula; entries include DEPT info where available.

Create `src/lucy_ng/dereplication/matcher.py` with:

```python
from enum import Enum
from dataclasses import dataclass, field

class MatchMode(Enum):
    """Matching mode based on available data."""
    SHIFTS_ONLY = "shifts_only"      # Just chemical shifts
    DEPT_ENHANCED = "dept_enhanced"  # Shifts + hydrogen count from DEPT

@dataclass
class MatchingConfig:
    """Configuration for spectrum matching."""
    mode: MatchMode = MatchMode.SHIFTS_ONLY
    tolerance_ppm: float = 1.0        # Default tolerance
    use_variable_tolerance: bool = True  # Region-dependent tolerances
    aliphatic_tolerance: float = 0.8  # 0-80 ppm
    aromatic_tolerance: float = 1.2   # 100-160 ppm
    carbonyl_tolerance: float = 1.5   # 160-220 ppm
    dept_mismatch_penalty: float = 0.5  # Score multiplier when DEPT doesn't match

    def get_tolerance(self, shift: float) -> float:
        """Get tolerance for a given chemical shift region."""
        if not self.use_variable_tolerance:
            return self.tolerance_ppm
        if shift < 80:
            return self.aliphatic_tolerance
        elif shift < 100:
            return self.tolerance_ppm  # Transition region
        elif shift < 160:
            return self.aromatic_tolerance
        else:
            return self.carbonyl_tolerance

@dataclass
class PeakMatch:
    """Details of a single peak match."""
    observed_shift: float
    reference_shift: float
    deviation: float  # absolute difference in ppm
    observed_h_count: HydrogenCount | None = None
    reference_h_count: HydrogenCount | None = None
    dept_match: bool | None = None  # None if DEPT not available

@dataclass
class MatchResult:
    """Result of matching observed peaks against a reference."""
    entry: NMRShiftDBEntry
    score: float  # 0-1, higher is better match
    matched_peaks: int
    total_observed: int
    total_reference: int
    expected_carbons: int  # from molecular formula
    dept_matches: int | None = None  # count of peaks with matching DEPT
    peak_matches: list[PeakMatch] = field(default_factory=list)
    unmatched_observed: list[float] = field(default_factory=list)
    unmatched_reference: list[float] = field(default_factory=list)

@dataclass
class ObservedPeak:
    """Observed peak with optional DEPT info for matching."""
    shift: float
    hydrogen_count: HydrogenCount | None = None

class SpectrumMatcher:
    """Match observed peaks against reference spectra with fuzzy tolerances."""

    def __init__(self, config: MatchingConfig | None = None):
        """Initialize with matching configuration."""
        self.config = config or MatchingConfig()

    def match(
        self,
        observed: list[ObservedPeak],
        reference: NMRShiftDBEntry,
    ) -> MatchResult:
        """Score how well observed peaks match a reference.

        Algorithm:
        1. For each observed peak, find closest reference peak within tolerance
        2. Track matched, unmatched observed, and unmatched reference peaks
        3. If DEPT mode, track how many matches have consistent hydrogen counts
        4. Calculate score accounting for expected overlap
        """

    def match_all(
        self,
        observed: list[ObservedPeak],
        references: list[NMRShiftDBEntry],
    ) -> list[MatchResult]:
        """Match against all references, return sorted by score descending."""

    def _find_best_match(
        self,
        observed_shift: float,
        available_refs: list[CarbonSignal],
    ) -> tuple[CarbonSignal | None, float]:
        """Find best matching reference peak within tolerance.

        Returns (matched_signal, deviation) or (None, inf) if no match.
        Uses region-dependent tolerance.
        """

    def _calculate_score(
        self,
        matched_peaks: int,
        observed_peaks: int,
        reference_peaks: int,
        expected_carbons: int,
        dept_matches: int | None,
    ) -> float:
        """Calculate match score accounting for expected overlap.

        The overlap_factor adjusts for the fact that observed peaks
        are often fewer than expected carbons due to signal overlap.
        """
```

Implementation notes:
- Use greedy matching: for each observed peak (sorted by shift), find closest unmatched reference
- Variable tolerance by chemical shift region (aliphatic tighter, carbonyl looser)
- Track detailed match info for diagnostics and debugging
- In DEPT mode: boost score when hydrogen counts match, penalize mismatches
- Score calculation uses overlap factor from molecular formula carbon count
- Return unmatched peaks for user inspection (potential impurities or missing assignments)

**Acceptance**: Can match peaks with configurable modes and tolerances; Ibuprofen matches correctly with DEPT enhancement.

## Research Notes

### Database Choice

nmrshiftdb API was considered but doesn't support molecular formula search directly. Using local SD file provides:
- Offline operation
- Fast queries after initial parse
- Full control over matching algorithm

### Matching Algorithm Design

The scoring algorithm went through several iterations:

1. **Naive**: `matched/observed × matched/reference` — penalizes overlap unfairly
2. **Overlap-adjusted**: Uses molecular formula carbon count to estimate expected overlap
3. **DEPT-enhanced**: Adds hydrogen count matching as score multiplier

Key insight: In real spectra, `observed_peaks ≤ expected_carbons` due to:
- Chemical equivalence (symmetric molecules)
- Accidental degeneracy (different carbons at same shift)
- Missing quaternary signals

The overlap factor `min(1.0, observed/expected)` adjusts reference penalty accordingly.

### Chemical Shift Regions

Tolerances tuned by region based on typical shift variability:
- **Aliphatic (0-80 ppm)**: Most consistent, tight tolerance (0.8 ppm)
- **Aromatic (100-160 ppm)**: Sensitive to substituent effects, medium tolerance (1.2 ppm)
- **Carbonyl (160-220 ppm)**: Most variable, loose tolerance (1.5 ppm)

### Future Enhancements (Out of Scope)

- Weighted tolerance by confidence (higher intensity peaks → tighter tolerance)
- Learning optimal tolerances from training data
- Substructure-aware matching (use connectivity for disambiguation)
- Multiple spectra combination (1H + 13C + DEPT-135 + DEPT-90)
