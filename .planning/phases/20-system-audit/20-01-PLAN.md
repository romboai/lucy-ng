  <name>Task 1: Audit all 15 MCP tools -- classify by intelligence tier</name>
  <files>.planning/phases/20-system-audit/audit-mcp-tools.md</files>
  <action>
Read `src/lucy_ng/mcp/server.py` (1,282 lines). For EACH of the 15 MCP tools (identified by `@mcp.tool()` decorator):

1. Record the tool name and its function signature
2. Trace the call chain: MCP tool -> Python module -> implementation class/function
3. Read the IMPLEMENTATION code (not just the MCP wrapper). For example:
   - `pick_hsqc_peaks` MCP tool -> `DEPTGuidedPicker.pick_hsqc_peaks` (289 lines in `processing/dept_guided_picker.py`)
   - `generate_lsd_input` MCP tool -> `LSDInputGenerator` (478 lines in `lsd/generator.py`)
4. Classify at the IMPLEMENTATION level using these criteria:

   **Tier 1 (pure data access -- keep as-is):**
   - Single function call to library (BrukerReader, DatabaseManager)
   - No conditional logic beyond error handling
   - Returns raw data or simple transformations

   **Tier 2 (moderate intelligence -- document strategy in skill):**
   - Default parameter values embedding domain knowledge (thresholds, tolerances)
   - Strategy selection or simple filtering rules
   - Standard published algorithms (SciPy peak finding, noise estimation)

   **Tier 3 (complex logic -- refactor to thin wrapper, move logic to skill):**
   - Multi-step algorithms with domain inference
   - Cross-validation between data sources
   - Heuristic-based decisions (carbonyl shift range -> sp2 oxygen pairing)

5. For each tool, write ONE specific recommendation (not generic). Examples:
   - GOOD: "Keep as-is. Pure wrapper around BrukerReader.read_1d()."
   - GOOD: "Document adaptive threshold strategy (start at 0.05, reduce until all DEPT carbons matched) in SKILL.md. Keep algorithm in Python."
   - GOOD: "Refactor: Accept pre-constructed atom definitions + constraints. Move carbonyl detection (160-220 ppm -> sp2 oxygen) and heteroatom pairing logic to SKILL.md."
   - BAD: "Simplify later" (too vague)

Write the output as a markdown file with this structure:

```markdown

## Summary

- Tier 2 (document strategy): N tools
- Tier 3 (refactor): N tools

## Tool Classifications

- **Implementation:** src/lucy_ng/module/file.py -> ClassName.method() (N lines)
- **Intelligence detected:** [specific description of domain logic found, or "None"]
- **Recommendation:** [specific actionable recommendation]

[repeat for all 15 tools]
```

IMPORTANT: Do NOT just classify the 20-line MCP wrappers. The intelligence lives in the implementation modules. You MUST read each implementation file to classify correctly.
