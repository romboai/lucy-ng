  <name>Task 1: Compile audit report with cross-cutting analysis and migration roadmap</name>
  <files>.planning/phases/20-system-audit/AUDIT-REPORT.md</files>
  <action>
Read the three audit section files created by Plans 01 and 02:
- `audit-mcp-tools.md` (MCP tool classifications)
- `audit-cli-commands.md` (CLI command classifications)
- `audit-claude-md.md` (CLAUDE.md section analysis)

Compile into a single `AUDIT-REPORT.md` with these sections:

**1. Executive Summary**
- Total components audited (15 MCP tools, 9 CLI groups / 22 commands, N CLAUDE.md sections)
- Tier distribution across all components (how many Tier 1/2/3)
- Top-line finding: what percentage of intelligence is in code vs. should be in skill
- Key duplication findings from CLAUDE.md

**2. MCP Tool Classifications** (from audit-mcp-tools.md)
- Include the full classification table
- Highlight the Tier 3 tools that need refactoring (these are Phase 26 work items)

**3. CLI Command Classifications** (from audit-cli-commands.md)
- Include the full classification table
- Note CLI/MCP overlap (shared implementations)

**4. CLAUDE.md Analysis** (from audit-claude-md.md)
- Include the section catalogue
- Include the duplication map
- Include the misplacement analysis

**5. Cross-Cutting Analysis**
This is NEW analysis not in the section files. Identify:
- **Intelligence hotspots**: Which Python modules contain the most domain logic? (e.g., DEPTGuidedPicker, HMBCGuidedPicker, LSDInputGenerator)
- **Shared implementations**: Where do MCP and CLI share the same underlying code? (important for Phase 26 dual-mode approach)
- **Skill document outline**: Based on all three audits, what should SKILL.md contain? (This feeds Phase 21 directly)

**6. Migration Roadmap**
Map each audit finding to a specific v2.0 phase:

| Component | Current State | Target State | Phase |
|-----------|--------------|-------------|-------|
| DEPTGuidedPicker | Tier 3: adaptive algorithm in Python | Strategy in SKILL.md, thin wrapper | Phase 26 |
| LSD Integration section | 360 lines duplicated in CLAUDE.md | Single section in SKILL.md | Phase 21 |
| [etc.] | | | |

Group by phase:
- **Phase 21 (Skill Restructure):** Which CLAUDE.md sections move where
- **Phase 22 (HMBC Strategy):** Which tool intelligence becomes skill content about HMBC
- **Phase 23 (Error Tolerance):** Which pitfall/error handling content becomes skill content
- **Phase 24 (Supervisor):** Which decision tree content feeds supervisor design
- **Phase 26 (Thin Tools):** Which Tier 3 tools get refactored, what their thin versions look like

**7. Recommendations Summary**
One-liner per component: "Component X: [action verb] [specific change]"

Example format:
```
- read_spectrum_1d (MCP): Keep as-is. Pure data wrapper.
- pick_hsqc_peaks (MCP): Document adaptive threshold strategy in SKILL.md (Phase 22). Keep algorithm in Python.
- generate_lsd_input (MCP): Refactor to accept pre-built constraints (Phase 26). Move heteroatom inference to SKILL.md (Phase 22).
- LSD Integration (CLAUDE.md): Merge with Manual LSD Construction into single SKILL.md section (Phase 21).
- Decision Trees (CLAUDE.md): Move to SKILL.md workflow section (Phase 21). Expand with incremental HMBC strategy (Phase 22).
```

IMPORTANT: Every recommendation must reference a specific v2.0 phase. No orphan recommendations.
