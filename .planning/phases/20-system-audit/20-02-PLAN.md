**Analysis 1: Section Catalogue**

For each top-level section (## heading) and significant subsection (### heading):
- Record heading, line range (start-end), approximate line count
- Categorize as one of:
  - **project-setup**: Installation, dependencies, permissions (stays in CLAUDE.md)

  - **case-workflow**: Structure elucidation steps, decision trees (moves to SKILL.md)

  - **tool-usage**: CLI/MCP usage examples, output format reference (split: keep brief reference in CLAUDE.md, detailed workflow in SKILL.md)

Research identified these duplication candidates -- verify and quantify:

For each duplication cluster:
- Quote the specific repeated concepts (not just "LSD stuff")
- Identify which instance is the canonical/complete version
- Identify which instances are abbreviated/redundant
- Recommend: keep canonical version in one place, remove duplicates

**Analysis 3: Misplaced Intelligence**

Flag any section where the content teaches the AI HOW to reason about NMR/CASE rather than how to use the tools. These belong in SKILL.md, not CLAUDE.md.

Examples of misplaced intelligence:
- Decision trees that encode CASE workflow logic
- "Common Pitfalls" that teach the AI what to watch for
- Interpretation guidelines (how to read dereplication scores, MAE scores)
- "When to use" guidance for different approaches

Examples of correctly placed content:
- Installation instructions
- CLI command reference (syntax, flags)
- Project structure and developer reference
- Build/test commands
