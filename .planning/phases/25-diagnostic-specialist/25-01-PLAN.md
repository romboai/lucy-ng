**Section 1: LSD Command Reference (DIAG-05)**
Complete reference for ALL LSD commands relevant to diagnostics. This goes DEEPER than skill/SKILL.md Section 6 -- include advanced commands:
- MULT: Full syntax with all element types (C, N, O, S), hybridization values (2=sp2, 3=sp3), hydrogen counts. Include edge cases: bridgehead carbons (sp3, 0H but not quaternary in classical sense), sp2 nitrogen in different ring types (pyridine vs pyrrole).
- HSQC/HMQC: Syntax, proton position definition semantics, ordering requirement.
- HMBC: Syntax, 2-3 bond distance semantics, common artifacts (1J leakthrough).
- BOND: Explicit bond constraint syntax, when to use vs LIST/PROP.
- LIST: Named atom list creation. Syntax: `LIST Ln atom1 atom2 ...`
- PROP: Property constraints on lists. Syntax: `PROP L1 count L2` (each atom in L1 has exactly `count` neighbors from L2). Show examples for heteroatom attachment, functional group constraints.
- ELEM: Element-type list creation. Syntax: `ELEM Ln element_symbol`. Creates list of all atoms of that element.
- SYME: Symmetry equivalence constraint. Syntax: `SYME atom1 atom2`. Forces atoms to be topologically equivalent. Note: may not be supported in all LSD versions; if unsupported, use LIST/PROP as fallback to encode symmetry constraints.
- DEFF: Deff angle constraint (rare, advanced). Brief mention for completeness.
- ELIM: Correlation elimination. Syntax: `ELIM P1 P2`. P1=max eliminations, P2=max bond distance (0=no limit). Emphasize: LAST RESORT ONLY.

Cross-reference skill/SKILL.md Section 6 -- do NOT duplicate the basic command format that is already there. Instead, focus on diagnostic-relevant details: what each command constrains, what happens when it's wrong, and how to detect errors in its usage.

**Section 2: Systematic Diagnostic Procedures**

**2.1 Zero-Solution Failure Procedure (DIAG-02)**
Ordered checklist that MUST be followed completely (all checks, not stop at first pass):

1. **sp2 Count Check**: Parse all MULT commands, count atoms with hybridization=2. MUST be even. If odd, identify which atom is likely wrong (common: ether oxygen marked sp2 instead of sp3, amine nitrogen marked sp2 instead of sp3). Provide fix examples with LSD MULT commands.

2. **Hydrogen Budget Check**: Sum hydrogen counts from all MULT commands. Compare to molecular formula. If mismatch, calculate difference and identify likely misassigned atoms (common: CH marked as quaternary, CH2 marked as CH). Provide fix examples.

3. **1J Artifact Detection**: Compare each HMBC correlation against all HSQC positions. Tolerance: +/-1.5 ppm (carbon), +/-0.3 ppm (proton). If match found, this is a 1J artifact -- a direct-bond correlation that leaked into the HMBC spectrum. Explain WHY this causes 0 solutions: LSD interprets HMBC as 2-3 bonds, but HSQC says 1 bond; this is impossible, so no solution exists. Provide fix: remove the offending HMBC command.

4. **Correlation Order Check**: Verify all HSQC commands appear before HMBC commands in the file. If HMBC references a proton position not yet defined by HSQC, LSD errors. Provide fix: reorder commands.

5. **Close Carbon Ambiguity Check**: Look for carbons within 3 ppm of each other. If HMBC correlations involve these carbons, the assignment may be wrong (e.g., HMBC assigned to carbon at 155.2 ppm but actually belongs to carbon at 153.8 ppm). Recommend LIST/PROP encoding for ambiguous assignments rather than definitive HMBC.

**2.2 Solution Explosion Procedure (DIAG-03)**
Ordered checklist for 1000+ solutions:

1. **ELIM Presence Check**: Search for ELIM command. If found, it is likely increasing the solution space. Recommend removal.

2. **Constraint/Atom Ratio**: Count MULT atoms and HMBC correlations. Calculate ratio. If < 0.5, insufficient constraints. Recommend adding high-confidence HMBC correlations following skill/SKILL.md Section 7 (incremental HMBC strategy). Provide target ratio guidance.

3. **Quaternary Carbon Connectivity**: Identify all quaternary carbons (MULT with 0 H, no HSQC). Count HMBC correlations involving each. Any quaternary with 0 HMBC = "floating atom" that massively increases solution space. Recommend targeted HMBC search or shift-based constraints.

4. **Heteroatom Position Constraints**: Count O, N, S atoms from MULT. Check for BOND or LIST/PROP constraints involving them. No constraints on heteroatoms = LSD tries all permutations. Recommend BOND for known positions, LIST/PROP for ambiguous. Provide examples for common functional groups (carbonyl, ether, amine, amide).

5. **Symmetry Encoding**: Check CASE-PROGRESS.md for symmetry detection notes. If symmetry detected but not encoded in LSD file, solution space is inflated. Recommend SYME command (or LIST/PROP fallback). Provide examples for para-substituted benzene, isopropyl, gem-dimethyl.
