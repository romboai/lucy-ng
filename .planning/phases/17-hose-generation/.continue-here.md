<decisions_made>

- **No explicit hydrogens**: All HOSE code operations must use molecules WITHOUT explicit H atoms. With explicit H: `C-4;HHHC(//)`, without: `C-4;C(//)`. These don't match!
- **1-based to 0-based conversion**: COCONUT CNMR_SHIFTS field uses 1-based atom indices. Convert with `atom_idx - 1` for RDKit.
- **SDF-based generation**: Read mol objects directly from SDF rather than parsing SMILES from database. This preserves correct atom ordering.
</decisions_made>

<context>
We discovered that the previous HOSE stats generation (completed overnight) produced wrong predictions because:
1. COCONUT uses 1-based atom indices in CNMR_SHIFTS field
2. The original code didn't account for this, causing ~25% atom index mismatches

The fix was to create SDFHOSEStatsGenerator that:
1. Reads mol objects directly from SDF (preserves atom order)
2. Converts 1-based indices to 0-based when parsing CNMR_SHIFTS
3. Generates HOSE codes from molecules without explicit H (consistent with predictor)
