  <behavior>
    The signal grouping module provides:

    1. **is_multiplicity_compatible(mult1, mult2) -> bool**
       - Same multiplicity -> True (CH2+CH2, CH+CH)
       - Both ambiguous (CH, CH3, CH/CH3) -> True
       - Different definite multiplicities -> False (CH2+CH, CH2+CH3)
       - Either None -> True (conservative)

    2. **group_signals(shifts, multiplicities=None, tolerance=0.25) -> GroupingResult**
       - Sorts by shift value
       - Uses complete linkage: all pairwise distances in cluster <= tolerance
       - Filters by multiplicity compatibility (splits incompatible into singletons)
       - Returns GroupingResult with groups + ungrouped indices + warnings

    3. **Pydantic models** (add to existing models.py):
       - `SignalGroup(BaseModel)`: indices (0-based), shifts, multiplicities, span, centroid
         - `atom_ids` property: 1-based LSD atom IDs
         - `lsd_atom_list()` method: returns "(2 3)" or "2" for single
       - `GroupingResult(BaseModel)`: tolerance, groups, ungrouped, total_signals, warnings
         - `summary()` method: human-readable text output

    Test cases (input -> expected output):
    - Empty list -> no groups, no ungrouped
    - [44.90, 45.03] (0.13 ppm apart) -> 1 group of 2, tolerance 0.25
    - [44.90, 45.03, 129.38] -> 1 group of 2 (44.90+45.03), 1 ungrouped (129.38)
    - [44.90, 45.03, 45.20] with mults [CH2, CH, CH3] -> 0 groups, 3 ungrouped (all incompatible)
    - [44.90, 45.03, 45.20] with mults [CH, CH/CH3, CH3] -> 1 group of 3 (all CH/CH3-compatible)
    - [10.0, 10.1, 10.2, 10.5] tolerance 0.25 -> group [10.0,10.1,10.2] (complete linkage: 0.2<=0.25) + ungrouped [10.5] (distance from 10.0 is 0.5)
    - [10.0, 10.2, 10.4, 10.6] tolerance 0.25 -> groups [10.0,10.2] + [10.4,10.6] (complete linkage prevents chaining)
    - All N signals in one group -> warning "Unusually large group"
    - lsd_atom_list() for indices [0,1] -> "(1 2)"
    - lsd_atom_list() for indices [3] -> "4"
  </behavior>
  <implementation>
    1. Add SignalGroup and GroupingResult Pydantic models to models.py (append after HHBResult class)
    2. Create grouping.py with is_multiplicity_compatible() and group_signals()
    3. Update __init__.py to export GroupingResult and group_signals
    4. Algorithm for group_signals():
       - Sort indices by shift value
       - Iterate sorted: try to add each point to current group using complete linkage check (all pairwise distances <= tolerance)
       - If cannot join, finalize current group, start new
       - After distance clustering: filter each multi-element group by multiplicity compatibility
       - If any pair in group is incompatible, split entire group into singletons (don't try subgrouping — complexity not justified)
       - Check for warning conditions: any group has > 50% of total signals

    IMPORTANT: Do NOT use numpy for centroid calculation — use statistics.mean() or sum()/len() to avoid adding numpy dependency for trivial math. The detection module (grouping.py) does NOT currently import numpy and should not need to.
  </implementation>
</feature>
