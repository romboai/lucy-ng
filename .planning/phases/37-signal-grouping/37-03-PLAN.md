  <action>
    Add `grouping` subcommand to the existing `analyze` click group in `analyze.py`, following the pattern of the existing `symmetry` subcommand.

    Command signature:
    ```
    lucy analyze grouping SHIFTS [--multiplicities/-m MULTS] [--tolerance/-t FLOAT] [--format text|json]
    ```

    Arguments:
    - SHIFTS: comma-separated chemical shifts (e.g., "44.90,45.03,129.38,127.26")

    Options:
    - --multiplicities/-m: comma-separated DEPT multiplicities (CH, CH2, CH3, CH/CH3). Must match shift count.
    - --tolerance/-t: distance tolerance in ppm (default: 0.25)
    - --format: output format, text (default) or json

    Implementation:
    1. Parse shifts from comma-separated string to list[float]
    2. Parse multiplicities from comma-separated string to list[str | None] (if provided)
    3. Validate multiplicity count matches shift count
    4. Call `group_signals()` from `lucy_ng.detection.grouping`
    5. Output result using GroupingResult.summary() for text or .model_dump_json() for JSON

    Text output format (following existing analyze symmetry pattern):
    ```
    Signal Grouping (tolerance: 0.25 ppm)
    Total signals: 4
    Groups found: 1
    Ungrouped: 2

    Groups:
      Group 1: (1 2)    <-- LSD atom list format
        Atom 1: 44.90 ppm (CH2)
        Atom 2: 45.03 ppm (CH2)
        Span: 0.130 ppm, Centroid: 44.97 ppm

    Ungrouped (single atoms):
      Atom 3: 129.38 ppm (CH)
      Atom 4: 127.26 ppm (CH)

    Note: Close shifts may represent truly different carbons.
    Verify by checking DEPT multiplicities and HMBC connectivity.
    ```

    JSON output: Use GroupingResult.model_dump_json(indent=2) directly.

    IMPORTANT: The text output MUST include a warning about false positive risk at the bottom:
    "Note: Close shifts may represent truly different carbons."
    This satisfies roadmap success criterion #5.

    Use lazy import for `group_signals` (import inside function body) following the pattern used in detect.py commands.
  </action>
