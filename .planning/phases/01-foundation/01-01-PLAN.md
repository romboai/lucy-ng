# Phase 1: Foundation — Plan 01

## Objective

Set up the Python package structure, dependencies, core data models, and test infrastructure for lucy-ng.

## Context

**Project**: lucy-ng — AI-agent powered CASE for organic natural products from NMR data
**Phase Goal**: Project structure, dependencies, and NMR library evaluation

**Key Decision**: Use **nmrglue** for NMR file parsing
- Most mature Python NMR library with academic citations
- Native Bruker 1D and 2D file support
- Built-in peak picking algorithms
- NumPy-based data representation
- BSD license (open source compatible)

## Tasks

### Task 1: Create Python Package Structure

Set up modern Python package with pyproject.toml.


### Task 2: Create Core Data Models

Define Pydantic models for NMR data structures.

**Spectrum1D model:**
- `data`: numpy array (spectral intensities)
- `ppm_scale`: numpy array (chemical shift axis)
- `nucleus`: str (e.g., "1H", "13C")
- `frequency`: float (spectrometer frequency in MHz)
- `solvent`: Optional[str]
- `metadata`: dict (acquisition parameters)

**Spectrum2D model:**
- `data`: 2D numpy array
- `f1_ppm_scale`: numpy array (indirect dimension)
- `f2_ppm_scale`: numpy array (direct dimension)
- `f1_nucleus`: str
- `f2_nucleus`: str
- `experiment_type`: str (e.g., "HSQC", "HMBC", "COSY")
- `frequency`: float
- `metadata`: dict

**Peak model:**
- `position`: float (ppm) or tuple[float, float] (2D)
- `intensity`: float
- `assignment`: Optional[str]
- `multiplicity`: Optional[str] (for 1D 1H)

**PeakList model:**
- `peaks`: list[Peak]
- `spectrum_type`: str
- `nucleus`: str or tuple[str, str]

**Verification**: Models can be instantiated and serialized to JSON
