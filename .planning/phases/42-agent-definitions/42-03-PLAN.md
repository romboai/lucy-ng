3. `<domain_knowledge>` section (~100-150 lines):

   **Ranking Algorithm (from Section 5, ~20 lines):**
   - Two-tier ranking: (1) match count descending, (2) MAE ascending
   - Why match count first: prevents wrong solutions with coincidentally low MAE from outranking correct solutions with better spectral coverage
   - Always report BOTH metrics when discussing results
   - Quality tiers based on matched count: full match = high confidence, 80%+ = medium, <80% = low

   **CLI Reference — lucy lsd rank (~25 lines):**
   - Syntax: `lucy lsd rank <solutions.smi> --shifts "<comma_separated_shifts>"`
   - Output fields: rank, smiles, matched_count, mae, quality, deviations
   - JSON output: `--format json`
   - Interpretation guide:
     - matched_count = N/M: how many predicted shifts match experimental within tolerance
     - MAE: mean absolute error in ppm across all predicted shifts
     - quality: "excellent" (<2 ppm MAE, full match), "good" (<3.5 ppm), "fair" (<5 ppm), "poor" (>5 ppm)
   - Ranking workflow: `lucy lsd rank analysis/iteration_NN/solutions.smi --shifts "155.08,151.58,..."`

   **CLI Reference — lucy predict c13 (~15 lines):**
   - Syntax: `lucy predict c13 "<SMILES>" --format json`
   - Output fields: predictions (atom_index, shift, confidence), success
   - Use for: independent verification of top-ranked solutions
   - Workflow: predict shifts for top 3-5 candidates, compare with experimental spectrum
   - Interpretation: each predicted shift has a confidence (High/Medium/Low based on HOSE code match quality)

   **Chemical Plausibility Assessment (~25 lines — new content):**
   - Check 1: Functional group consistency — do predicted functional groups match the shift evidence?
   - Check 2: Degree of unsaturation — does the top structure's DBE match the molecular formula?
   - Check 3: Strain energy — are there unusual ring sizes or strained geometries?
   - Check 4: Spectral fit — are there systematic deviations suggesting wrong stereochemistry or regiochemistry?
   - Check 5: Natural product likelihood — is the structure plausible as a natural product (no bizarre heteroatom arrangements)?
   - Report format: for each top candidate, list plausibility assessment as PLAUSIBLE / QUESTIONABLE / IMPLAUSIBLE with reasoning

   **Confidence Scoring (from Section 7, ~35 lines — shared with nmr-chemist):**
   - Per-atom confidence: 3-factor model
     - Factor 1: Digital resolution (High/Medium/Low based on peak isolation)
     - Factor 2: HOSE prediction quality (MAE <2=High, 2-3.5=Medium, >3.5=Low)
     - Factor 3: Supporting correlations (3+ HMBC=High, 1-2=Medium, 0=Low)
   - Explicit downgrade rules (5 rules):
     1. Any ambiguity -> at most Medium
     2. MAE > 3.5 ppm for any atom -> that atom Low
     3. 0 HMBC on quaternary -> that atom Low
     4. DEPT/HSQC conflict unresolved -> that atom Medium at best
     5. Targeted threshold reduction failed -> quaternary Low
   - Per-structure confidence derivation:
     - High: >=80% atoms High/Medium, <=1 Low (non-critical)
     - Medium: >=50% High/Medium, or 2-3 Low (non-critical)
     - Low: <50% High/Medium, or 3+ Low, or critical atoms Low
   - "Err on the side of honesty"

   **Final Results Report Format (~15 lines):**
   - Write to: analysis/final_results.md
   - Sections: Top Candidates (ranked table), Confidence Assessment, Chemical Plausibility, Ambiguities and Caveats, Recommendation
   - Ranked table columns: Rank, SMILES, Matched, MAE, Quality, Plausibility
   - Always include: "Structure determination confidence: {High/Medium/Low}" as final line

4. `<message_interface>` section (~12 lines):

   **OUTPUTS:**
   - Ranked solutions table (rank, SMILES, MAE, matched count, quality)

   - Confidence assessment (per-atom and per-structure)
   - Final results summary with recommendation

   **INPUTS:**

   - From nmr-chemist: experimental shift list for ranking, detection results for confidence
   - From orchestrator: task assignments

5. `<workflow>` section (~12 lines):
   - Step 1: Claim ranking task from TaskList
   - Step 2: Read solutions.smi path and experimental shifts from task/messages
   - Step 3: Run ranking: `lucy lsd rank <solutions.smi> --shifts "<shifts>"`

   - Step 6: Compute confidence scores (per-atom and per-structure)
   - Step 7: Write analysis/final_results.md

   - Step 9: Mark task completed via TaskUpdate
