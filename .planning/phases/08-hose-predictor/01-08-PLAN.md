# Phase 8: HOSE-Based 13C Spectrum Predictor

## Goal

Build a pure Python 13C NMR chemical shift predictor using HOSE codes and the COCONUT database, enabling spectrum prediction for candidate structures in the CASE pipeline.

## Background

### Why HOSE Codes?

Evaluation of GNN-based predictors (nmrgnn, nmr_mpnn_pytorch, CASCADE) revealed critical dependency issues:
- **nmrgnn**: Uses deprecated TensorFlow APIs, incompatible with TF 2.x
- **nmr_mpnn_pytorch**: No pre-trained models, training-only codebase
- **CASCADE**: Requires Python 3.5 and TensorFlow 1.12

HOSE (Hierarchically Ordered Spherical Environment) codes provide a stable, pure-Python alternative:
- Only depends on RDKit (already a lucy-ng dependency)
- Well-established method with ~1.4-2.2 ppm MAE for 13C
- No ML framework version conflicts

### Data Source

Using COCONUT predicted 13C database (~895,000 entries) instead of nmrshiftdb2 (~33,000):
- 27x more training data for better HOSE code coverage
- Already integrated via `CoconutLoader`
- Includes multiplicity information (C, CH, CH2, CH3)

## Implementation Plan

### Step 1: HOSE Code Generation Wrapper

Create a thin wrapper around the `hosegen` library for our use case.

```python
# src/lucy_ng/prediction/hose.py
from rdkit import Chem
from hosegen import HoseGenerator

class HOSECodeGenerator:
    """Generate HOSE codes for carbon atoms in molecules."""

    @staticmethod
    def generate_for_carbons(mol: Chem.Mol, max_radius: int = 6) -> dict[int, str]:
        """Generate HOSE codes for all carbon atoms.

        Args:
            mol: RDKit Mol object (with explicit Hs)
            max_radius: Number of spheres for HOSE code (default 6)

        Returns:
            Dict mapping atom index to HOSE code
        """
```

### Step 2: HOSE Lookup Table Builder

Build lookup table by streaming through COCONUT SD file.

```python
# src/lucy_ng/prediction/lookup.py
class HOSELookupTable:
    """HOSE code to chemical shift lookup table."""

    def __init__(self):
        self._table: dict[str, list[ShiftEntry]] = {}

    @classmethod
    def build_from_coconut(cls, sd_path: Path, max_radius: int = 6) -> "HOSELookupTable":
        """Build lookup table from COCONUT SD file.

        Streams through file to avoid memory issues with 4.8GB file.
        Generates HOSE codes at all radii from 1 to max_radius for fallback.
        """

    def lookup(self, hose_code: str) -> list[float]:
        """Get all known shifts for a HOSE code."""

    def save(self, path: Path) -> None:
        """Save lookup table to JSON for fast reload."""

    @classmethod
    def load(cls, path: Path) -> "HOSELookupTable":
        """Load pre-built lookup table from JSON."""
```

### Step 3: Predictor Class

Main prediction interface.

```python
# src/lucy_ng/prediction/predictor.py
class C13Predictor:
    """Predict 13C NMR chemical shifts from structure."""

    def __init__(self, lookup_table: HOSELookupTable):
        self._lookup = lookup_table
        self._hose_gen = HOSECodeGenerator()

    def predict_from_smiles(self, smiles: str) -> list[PredictedShift]:
        """Predict 13C shifts for a molecule from SMILES.

        Returns:
            List of PredictedShift with atom_idx, shift, confidence, hose_code
        """

    def predict_from_mol(self, mol: Chem.Mol) -> list[PredictedShift]:
        """Predict from RDKit Mol object."""
```

### Step 4: Fallback Strategy

Handle cases where exact HOSE code isn't in lookup table:

1. Try exact match at max_radius (6 spheres)
2. If no match, try progressively shorter radii (5 → 4 → 3 → 2 → 1)
3. Return median of matches with confidence based on:
   - Number of matches found
   - Radius at which match was found (higher = more confident)
   - Standard deviation of matched shifts (lower = more confident)

class PredictedShift(BaseModel):
    """A predicted 13C chemical shift."""
    atom_index: int

## Performance Considerations

1. **Lookup table size**: ~895K molecules × ~15 carbons avg × 6 radii = ~80M HOSE entries
   - JSON storage: estimate 1-2GB
   - Consider msgpack or sqlite for faster loading/querying

2. **Build time**: One-time process, estimate 1-2 hours for COCONUT
   - Progress bar with tqdm
   - Resumable if interrupted

3. **Prediction speed**: O(n) where n = carbons in query molecule
   - Dict lookup is O(1)
   - Should be <100ms per molecule

## Testing Strategy

1. **Unit tests**: HOSE generation, lookup table operations
2. **Integration tests**: Full prediction pipeline
3. **Validation**: Compare predictions vs known experimental shifts
   - Use nmrshiftdb2 as validation set (not in training data)
   - Report MAE, RMSE, R² metrics
