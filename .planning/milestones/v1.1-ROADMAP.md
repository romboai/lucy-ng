# Milestone v1.1: Database-Backed Dereplication

**Status:** SHIPPED 2026-01-15
**Phases:** 11-15
**Total Plans:** 5

## Overview

Enable efficient dereplication against large databases (COCONUT 895K compounds + NMRShiftDB 33K) using SQLite indexing by molecular formula. This milestone delivered ~100x faster dereplication compared to scanning SD files.

## Phases

### Phase 11: Database Schema

**Goal**: Design and implement SQLite schema for compounds and 13C shifts, indexed by molecular formula

- SQLite database schema for compounds (id, name, smiles, formula, source)
- Schema for 13C shifts (compound_id, atom_index, shift_ppm)
- Index on molecular formula for fast lookup
- Pydantic models for database entities

**Depends on**: Milestone 1.0 complete
**Plans**: 1 plan

## Milestone Summary

**Key Decisions:**
- SQLite for dereplication DB (Rationale: Portable, no server, formula-indexed)
- Default database location: data/reference/compounds.db
- Environment variable LUCY_DATABASE for custom locations
- Use Any type for loader abstraction to support multiple backends
- database_type field for agent transparency about backend used

**Issues Resolved:**
- Slow dereplication against large databases (SD file scanning was O(n))
- Now O(1) lookup by molecular formula via SQLite index

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- None significant
